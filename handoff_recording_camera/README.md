# Handoff（聚焦改动）：录音中页面 · 新增「拍照」键

## 改动范围（只做这一项）
在**录音进行中**的全屏页面，**停止键右侧的空白区**新增一个**很淡的「拍照」键**。其它一律不动——计时器、波形、停止键、布局、配色全部保持现状。

对应代码：录音全屏接管页（方案二），文件 `RecordSession.swift`。参考稿：`Navigation.dc.html` 第 ② 屏「录音中 · 大秒表」。

---

## 这一项具体要求

底部操作区是一条横向 bar，分三部分（从屏幕底边往上约 26pt 处）：

1. **停止键**：保持**整屏水平居中、绝对不变**。白色圆形 66×66，1px 描边 `#E8DECF`，柔影 `0 4px 14px rgba(0,0,0,0.06)`；内含红色圆角方块 26×26 / r6 / `#E5392E`。下方标签「点击停止」12pt / `#8A8175`。
2. **拍照键（本次新增）**：放在**停止键右侧空白区域的水平中点**（即从屏幕中线到右边缘这段区间的居中位置，约屏宽 75% 处），**纵向与停止键圆心对齐**。
   - 样式刻意**很淡、无容器**：没有底色、没有描边、没有投影——只是一个细线条相机图标。
   - 图标：相机轮廓，线宽 1.3，描边色 `#A89E8E`，整体 **opacity 0.45**，绘制尺寸约 24×24，点击热区 42×42。
   - 下方标签「拍照」11pt / `#C2B8A8` / 字距 2。
   - 点击 = 进入拍照（沿用现有 `capturePhoto` 隐藏拍照逻辑；录音不中断）。
3. 左侧无按钮（保持停止键真正居中，不要用左右对称占位把停止键推偏）。

### 实现要点
- 停止键居中务必稳定：用 ZStack / overlay，让停止键在容器中央，拍照键以 overlay(alignment) 或绝对偏移独立定位，**不要**用 HStack 三等分（会把停止键挤离中心）。
- 拍照键纵向对齐到**停止键圆心**（不是整列含标签的中心）：图标中心 = 停止圆心。
- 录音态下点击拍照**不停止录音**；拍完回到录音态。
